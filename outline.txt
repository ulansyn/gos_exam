Как отвечать на экзамене Contents
Самая удобная формула устного ответа:

Аныктама / определение -> как работает -> пример -> где используется

Например для Stack:

Stack - LIFO структура.
Элемент push менен кошулат, pop менен алынат.
Мисалы: тарелка стопкасы.
Ал рекурсияда, функция чакырууларында жана postfix эсептееде колдонулат.

1. Структуры данных: Stack жана Queue
Stack - стек
Stack - бул маалымат структурасы, анда акыркы кирген элемент биринчи чыгат.

Негизги принцип:

LIFO - Last In, First Out
Акыркы кошулган элемент биринчи алынат.

Мисал: тарелкалар стопкасы. Акыркы койгон тарелканы биринчи аласың.

Негизги операциялар:

Операция Мааниси
push стекке элемент кошуу
pop акыркы элементти алып салуу
peek/top эң үстүндөгү элементти көрүү
isEmpty стек бошпу текшерүү
Мисал:

push(10)
push(20)
push(30)

Stack: [10, 20, 30]

pop() => 30
pop() => 20

Стек кайда колдонулат?

Функция чакырууларында - call stack.
Infix -> Postfix айландырууда.
Скобкаларды текшерүүдө:{},(),[].
Undo/Redo операцияларында.
Рекурсияда.
Browser back history.
Устно жооп:

Стек - бул LIFO принциби менен иштеген структура. Элементтер бир жагынан гана кошулат
жана алынат. Негизги операциялары push, pop, peek. Ал функция чакырууларында,
рекурсияда, скобка текшерүүдө жана infix-postfix алгоритмдеринде колдонулат.
Queue - кезек
Queue - бул маалымат структурасы, анда биринчи кирген элемент биринчи чыгат.

Принцип:

FIFO - First In, First Out
Биринчи келген биринчи чыгат.

Мисал: банктагы кезек.

Негизги операциялар:

Операция Мааниси
enqueue кезекке кошуу
dequeue кезектен алып салуу
front биринчи элементти көрүү
rear акыркы элемент
isEmpty бошпу текшерүү
Мисал:

enqueue(10)
enqueue(20)
enqueue(30)

Queue: [10, 20, 30]

dequeue() => 10
dequeue() => 20

Queue кайда колдонулат?

Принтер тапшырмаларында.
Процессор scheduling.
Network packet processing.
BFS алгоритминде.
Message queue системаларында.
Customer service системаларында.
Устно жооп:

Queue - FIFO принциби менен иштеген структура. Биринчи кошулган элемент биринчи
алынат. Негизги операциялары enqueue жана dequeue. Queue процессор пландаштырууда,
тармак пакеттерин иштетүүдө, BFS алгоритминде жана принтер кезектеринде колдонулат.
Stack vs Queue
Белги Stack Queue
Принцип LIFO FIFO
Кошуу үстүнө артына
Алуу үстүнөн башынан
Мисал тарелка стопкасы адамдар кезеги
Операциялар push/pop enqueue/dequeue
2. Infix жана Postfix
Infix деген эмне?.
Infix - оператор эки операнддын ортосунда жазылат.

Мисал:

A + B
(A + B) * C

Бул биз күнүмдүк колдонгон форма.

Postfix деген эмне?
Postfix же Reverse Polish Notation - оператор операнддардан кийин жазылат.

Мисал:

A B +
A B + C *

Мисалы:

Infix: A + B
Postfix: A B +

Infix: (A + B) * C
Postfix: A B + C *

Эмне үчүн postfix керек?
Компьютерге postfix эсептөө оңой, анткени скобка жана приоритетти текшерүү жеңил болот. Postfix
көбүнчө stack аркылуу эсептелет.

Postfix эсептөө мисалы
Берилди:

5 3 + 2 *

Кадамдар:

5 push
3 push

=> 5 + 3 = 8
2 push
=> 8 * 2 = 16
Жооп: 16

Infix -> Postfix алгоритми кыскача
Stack колдонулат.

Операнд чыкса - outputка жаз.
(чыкса - stackке сал.
)чыкса -(чыкканга чейин stackтен чыгар.
Оператор чыкса - приоритетин салыштыр.
Аягында stackтеги операторлорду outputка чыгар.
Приоритет:

/ жогору
төмөн
Мисал:

Infix: A + B * C
Postfix: A B C * +

Анткени көбөйтүү биринчи аткарылат.

Устно жооп:

Infix - оператор операнддардын ортосунда жазылган форма, мисалы A+B. Postfix - оператор
операнддардан кийин жазылган форма, мисалы AB+. Postfixти эсептөө үчүн стек колдонулат.
Операнддар стекке салынат, оператор келгенде эки операнд алынып, операция аткарылып,
жыйынтык кайра стекке салынат.
3. База данных
База данных деген эмне?
База данных - бул маалыматтарды уюшкан түрдө сактоо, издөө, өзгөртүү жана башкаруу үчүн

банктагы кардарлар базасы;

DBMS деген эмне?.
DBMS - Database Management System
Кыргызча/орусча: СУБД - система управления базами данных.

СУБД маалымат базасын түзүүгө, сактоого, өзгөртүүгө жана суроо-талап кылууга жардам берет.

Мисалдар:

MySQL
PostgreSQL

Реляционная база данныхда маалыматтар таблицаларда сакталат.

Мисал:Students

id name group
1 Ali IT-
2 Aibek IT-
Негизги түшүнүктөр:

Термин Мааниси

SQL мисалдары
Маалымат алуу:

SELECT * FROM students;

Кошуу:

INSERT INTO students (name, group_name)
VALUES ('Ali', 'IT-1');

Өзгөртүү:

UPDATE students
SET group_name = 'IT-2'
WHERE id = 1 ;

Өчүрүү:

DELETE FROM students
WHERE id = 1 ;

Нормализация.
Нормализация - маалыматтарды кайталанууну азайтып, туура таблицаларга бөлүү процесси.

Максаты:

маалымат кайталанбасын;
жаңылыштык азайсын;
база иреттүү болсун;
байланыштар туура түзүлсүн.
Транзакция
Транзакция - бир бүтүн операциялар тобу.

Мисал: банктык которуу.

Асан эсебинен 100 сом кемийт.
Үсөн эсебине 100 сом кошулат.
Экөө тең аткарылышы керек. Бирөө аткарылып, экинчиси аткарылбай калса - ката.

ACID
Принцип Мааниси
Atomicity бүт операция толук аткарылат же такыр аткарылбайт
Consistency база туура абалда калат
Isolation транзакциялар бири-бирине тоскоол болбойт
Durability аткарылган өзгөрүү сакталып калат
Устно жооп:

База данных - маалыматтарды уюшкан түрдө сактоо жана башкаруу системасы. Аны СУБД
башкарат. Реляциялык базада маалымат таблицаларда сакталат. Негизги түшүнүктөр:
таблица, жазуу, талаа, primary key, foreign key жана SQL. Маалыматтын бүтүндүгүн сактоо
үчүн нормализация жана транзакциялар колдонулат.

7. Protocols жана Port Numbers
Protocol деген эмне?
Protocol - тармактагы түзмөктөр маалымат алмашууда карманган эрежелер жыйындысы.

Мисалдар:

Протокол Вазифасы Порт

HTTPS коопсуз веб 443
FTP файл берүү 20/21
SSH алыстан башкаруу 22

SMTP email жөнөтүү 25

IMAP email алуу/синхрондоо 143

Port number деген эмне?
Port - компьютердеги белгилүү бир кызматты аныктаган номер.

IP дарек компьютерди табат, ал эми port ошол компьютердеги кызматты табат.

Мисал:

192.168.1.10:80
Бул жерде:

192.168.1.10 = IP address
80 = HTTP service port

Port түрлөрү
Диапазон Аталышы Мааниси
0-1023 Well-known ports белгилүү кызматтар
1024-49151 Registered ports тиркемелер үчүн катталган
49152-65535 Dynamic/Ephemeral ports убактылуу порттор
Dynamic port number
Dynamic port же ephemeral port - клиент тарабынан убактылуу колдонулган порт.

Мисал:

Сен браузерден сайт ачасың:

Client: 192.168.1.5:53000
Server: 172.217.1.1:443

Бул жерде:

53000 = dynamic client port
443 = HTTPS server port

Устно жооп:

Port number - бул компьютердеги белгилүү кызматты аныктаган номер. Мисалы HTTP 80,
HTTPS 443, DNS 53 портун колдонот. Dynamic port - клиент тарабынан убактылуу берилген
порт, көбүнчө 49152ден 65535ке чейинки диапазондо болот.

12. Software Requirements - программалык талаптар

16. Memory Fragmentation
Fragmentation деген эмне?
Memory fragmentation - эс тутум майда бош бөлүктөргө бөлүнүп, чоң блок бөлүүгө кыйын болуп
калган абал.

Эки түрү бар.

1. Internal fragmentation
Бөлүнгөн memory ичинде колдонулбай калган бош орун.

Мисал:

Process 18 KB сурады
OS 20 KB block берди
2 KB колдонулбай калды

Бул - internal fragmentation.

2. External fragmentation
Memoryде бош орун бар, бирок ал майда бөлүктөргө бөлүнүп калган.

Мисал:

Бош орундар: 5 KB + 10 KB + 8 KB = 23 KB
Process 20 KB сурады
Бир бүтүн 20 KB блок жок

Бул - external fragmentation.

Чечүү жолдору.
Маселе Чечим
Internal fragmentation туура block size
External fragmentation compaction, paging, segmentation
Жалпы virtual memory
Устно жооп:

Fragmentation - эс тутумдун натыйжасыз колдонулушу. Internal fragmentation бөлүнгөн
блоктун ичинде бош орун калганда болот. External fragmentation бош орундар майда
бөлүктөргө бөлүнүп, чоң блок бөлүү мүмкүн болбогондо болот.
17. Virtual Memory
Virtual memory деген эмне?.
Virtual memory - операциялык система RAM жетишсиз болгондо дисктин бир бөлүгүн кошумча эс
тутум катары колдонгон механизм.

Бул процесс ар бир программага чоң жана үзгүлтүксүз memory бардай көрүнүш берет.

Эмне үчүн керек?
RAM жетишпегенде программаларды иштетүү;
процесстерди изоляциялоо;
memory management жеңилдетүү;
коопсуздукту жогорулатуу.
Page жана Frame
Virtual memory көбүнчө paging аркылуу иштейт.

Термин Мааниси
Page virtual memory бөлүгү
Frame physical RAM бөлүгү
Page table virtual address -> physical address которот
Page fault
Page fault - керек болгон page RAMда жок болгондо пайда болот. OS аны дисктен RAMга жүктөйт.

Swap
Swap - RAM жетишсиз болгондо айрым маалыматтарды дискке чыгаруу.

Устно жооп:

Virtual memory - RAM жетишсиз болгондо дисктин бир бөлүгүн эс тутум катары колдонгон
механизм. Ал программаларга чоң memory бардай көрүнүш берет. Virtual memory paging
аркылуу иштейт: virtual pages physical frames менен байланышат. Керектүү page RAMда жок
болсо, page fault болот.

20. Маанилүү кыска жооптор
Stack кандай иштейт?
Stack LIFO принциби менен иштейт. Акыркы кошулган элемент биринчи чыгат. Push элемент
кошот, pop элементти алып салат.
Queue кандай иштейт?
Queue FIFO принциби менен иштейт. Биринчи кошулган элемент биринчи чыгат. Enqueue
кошот, dequeue алып салат.
Process деген эмне?
Process - аткарылып жаткан программа, өзүнүн memory space жана ресурстары бар.
Thread деген эмне?
Thread - process ичиндеги аткаруу агымы, ал process ресурсарын бөлүшөт.
Virtual memory деген эмне?
Virtual memory - RAM жетишсиз болгондо дисктин бөлүгүн кошумча memory катары колдонуу
механизми.
Fragmentation деген эмне?
Fragmentation - memory бош орундардын майда бөлүктөргө бөлүнүп, натыйжасыз колдонулушу.
FAT деген эмне?
FAT - файлдардын дисктеги кластерлерин көрсөткөн File Allocation Table файл системасы.
PDU деген эмне?
PDU - ар бир OSI катмарындагы маалымат бирдиги: Data, Segment, Packet, Frame, Bits.
Encapsulation деген эмне?
Encapsulation - маалыматка ар бир тармак катмары өз headerин кошуу процесси.
Protocol деген эмне?
Protocol - тармактагы маалымат алмашуу эрежелери.
Port деген эмне?
Port - компьютердеги кызматты аныктаган номер.
Dynamic port деген эмне?
Dynamic port - клиент тарабынан убактылуу колдонулган порт, көбүнчө 49152-65535
диапазонунда.

22. Мини-экзамен: өзүңдү текшер
Суроо 1
Stack менен Queue айырмасы эмнеде?

Жооп:

Stack LIFO, Queue FIFO. Stackте акыркы кирген биринчи чыгат, Queueда биринчи кирген
биринчи чыгат.
Суроо 2
Infix жана Postfix айырмасы?

Жооп:

Infixте оператор операнддардын ортосунда жазылат: A+B. Postfixте оператор операнддардан
кийин жазылат: AB+.
Суроо 3
PDU деген эмне?

Жооп:

Protocol Data Unit - OSI катмарындагы маалымат бирдиги.
Суроо 4
TCP менен UDP айырмасы?

Жооп:

TCP ишенимдүү, connection-oriented, маалымат жеткенин текшерет. UDP connectionless, тез,
бирок ишенимдүүлүгү азыраак.
Суроо 5
Virtual memory эмне үчүн керек?

Жооп:

RAM жетишсиз болгондо дискти кошумча memory катары колдонуу жана процесстерге чоң
memory бардай көрсөтүү үчүн.
Суроо 6
Software requirement деген эмне?

Жооп:

Система эмне кылышы керек жана кандай сапатта иштеши керек экенин сүрөттөгөн талап.
Суроо 7
Маалымат системасы эмнелерден турат?

Жооп:

Hardware, software, data, people, procedures жана network компоненттеринен турат.
23. Кыскача экзамендик план
Эгер убакыт аз болсо, биринчи буларды жатта:

Stack / Queue - LIFO/FIFO, push/pop, enqueue/dequeue.
Infix/Postfix - стек менен эсептөө.
OSI model - 7 катмар жана PDU.
Protocols/Ports - HTTP 80, HTTPS 443, DNS 53, DHCP 67/68.
Process/Thread - айырмасы.
Virtual memory / Fragmentation / FAT.
Database - DBMS, table, key, SQL, transaction.
Software lifecycle - SDLC этаптары.
Requirements - functional жана non-functional.
Маалымат системасы - компоненттери.
Эң күчтүү устный жооп формуласы:

Аныктама -> кандай иштейт -> мисал -> кайда колдонулат

Мисалы Stack:

Stack - LIFO структура.
Элемент push менен кошулат, pop менен алынат.
Мисалы тарелка стопкасы.
Ал рекурсияда, функция чакырууларында жана postfix эсептөөдө колдонулат.
